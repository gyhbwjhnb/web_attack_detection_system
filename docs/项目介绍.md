# 常见网络攻击的检测系统

## 一、项目背景

随着互联网的高速发展，网络攻击手段日益多样化和复杂化。从传统的端口扫描、口令暴力破解，到 SQL 注入、XSS 跨站脚本攻击、木马远控、DDoS 等，网络安全威胁无处不在。入侵检测系统（Intrusion Detection System, IDS）作为网络安全防护体系的重要组成部分，能够实时监控网络流量，发现违反安全策略的行为并及时告警。

本项目旨在开发一个**基于特征串匹配和异常行为分析的轻量级网络攻击检测系统**，帮助小组成员深入理解入侵检测的基本原理与实现方法。

## 二、项目目标

开发一个具备以下能力的网络攻击检测系统：

1. **特征匹配检测**：内置攻击特征库，通过字符串匹配算法检测已知网络攻击（如 SQL 注入、XSS、Web 攻击、木马后门、恶意命令执行等）。
2. **异常行为检测**：建立正常网络行为基线，识别偏离基线的异常行为（如单 IP 高频扫描、短时间大量登录、异常外联陌生 IP、内网横向扩散等）。
3. **友好的图形用户界面**：提供实时告警展示、配置管理、日志查看等功能。

## 四、可检测的攻击类型

### 特征匹配覆盖范围
- SQL 注入攻击（如 `' OR 1=1 --`、`UNION SELECT` 等）
- XSS 跨站脚本攻击（如 `<script>` 标签注入）
- 暴力破解检测（如 FTP/SSH 多次登录失败）
- Web 攻击（目录遍历、命令注入、文件包含等）
- 木马/后门通信（已知 C2 通信特征）
- 漏洞利用载荷（如缓冲区溢出 shellcode 特征）

### 异常行为检测覆盖范围
- 端口扫描行为（单 IP 对多端口短时间大量连接）
- 暴力登录行为（短时间内大量认证请求）
- 异常外联行为（内网主机连接外部陌生 IP）
- 内网横向扩散（内网主机间异常访问模式）
- DDoS 流量特征（SYN Flood 等）
- ★ 智能增强：攻击置信度评分、24h 异常时段检测、自学习误报抑制

#### 智能检测增强（v2.0）
- **检测器插件化**：基于 `IDetector` 接口，支持运行时热插拔和自定义检测器扩展
- **规则热加载**：支持文件夹监控，拖放 JSON 规则文件即生效
- **置信度评分**：7 种攻击类型动态计算 0.0-1.0 置信度
- **异常时段检测**：24h EMA 基线，凌晨异常活动自动提权
- **自学习抑制**：用户标记误报后自动学习抑制规则，跨会话持久化

## 五、项目组织结构

项目划分为 **4 个独立模块**，由 4 名成员分别负责开发，最后集成联调。

| 模块编号 | 模块名称 | 负责人 | 主要职责 |
|---------|---------|--------|---------|
| 模块一 | 数据包捕获与预处理 | 成员 A | 网络数据包抓取、过滤、解析 |
| 模块二 | 特征匹配检测引擎 | 成员 B | 攻击特征库管理、字符串匹配检测 |
| 模块三 | 异常行为检测引擎 | 成员 C | 基线建立、异常行为统计分析 |
| 模块四 | 告警输出与图形界面 | 成员 D | 告警管理、GUI 展示、配置管理 |

## 六、开发计划建议

1. **第一阶段（模块独立开发）**：各成员依据《设计文档》中定义的公共接口，独立完成各自模块的编码与单元测试。
2. **第二阶段（集成联调）**：将四个模块集成到一个主程序中，通过公共数据结构进行数据传递，测试端到端流程。
3. **第三阶段（测试优化）**：使用测试 pcap 文件或模拟攻击流量进行整体测试，修复 bug，优化性能。
4. **第四阶段（文档与演示）**：编写总体设计报告，准备答辩演示。
